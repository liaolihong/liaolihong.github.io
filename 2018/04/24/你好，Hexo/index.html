你好，Hexo

发表于 2018-04-24 | 分类于 搭建博客
字数统计: 799 | 阅读时长 ≈ 3

今天，突然心血来潮，想自己搭建一个自己的静态网站，写写博客。在网上找了一下，发现目前比较实用的就是Hexo了。

搭建的过程中主要参考使用Hexo+Github一步步搭建属于自己的博客（基础） 搭建而成。在搭建的过程中，也踩了几次坑，所以记录一下，希望下次不要再掉进去了。

首先，搭建这样一个Hexo+Github的博客需要安装一些必备的软件。

1、安装node.js

刚开始我安装node的时候，用的是版本比较低的，导致后面都一直用npm安装不了依赖。所以还是安装高版本的比较好，我安装的就是8.11.1版本的。

安装好node之后配置一下环境变量（下次搭建的时候看看不配置环境变量会不会成功）。

2、安装git

git直接在官网上面下载，版本是2.16左右的就可以了。安装完右键发现有git gui here和git bash here就说明成功了。

3、Github账户新建项目的时候必须遵守格式：账户名.github.io，不然接下来会有很多麻烦，这个确实要遵守这个命名规则，在后面生成的github page网址的时候就是根据这个来的，如果名称不是按照这个规则做的，会导致访问不了，页面打开报404错误。

4、安装Hexo，在合适的地方新建一个文件夹存放该项目。
输入npm install hexo -g开始安装Hexo
输入hexo -v检查是否安装成功
输入hexo init,初始化该文件夹
看到start blogging with hexo就是初始化成功了
再输入npm install，安装需要的组件。

输入hexo g生成hexo
再输入hexo s，开启服务器，访问网址

5、将hexo和github联系起来
输入cd ~/.ssh,检查是否有.ssh文件夹
输入ls列出文件夹的内容
输入ssh-keygen -t rsa -C “你的邮箱”，连续三个回车，生成密钥，最后得到了两个文件：id_rsa和id_rsa.pub（默认存储路径是：C:\Users\Administrator.ssh）。
输入eval “$(ssh-agent -s)”，添加密钥到ssh-agent
再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent

此时，登录github点击头像下的settings，添加ssh。
输入ssh -T git@github.com，测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了

6、配置Deployment，在项目文件夹中，找到_config.yml文件，修改repo值

deploy:
type: git
repository: git@github.com:liaolihong/liaolihong.github.io.git（在github项目里的ssh（右下角））
branch: master

7、新建一篇博客，输入hexo new post “博客名”
在编写文章之前，需要安装扩展：npm install hexo-deployer-git –save
当编辑好博客之后，使用命令：hexo d -g生成以及部署到github上。部署成功后，访问http://用户名.github.io。那么将看到生成的文章。

此次搭建环境是第一次，感觉花费时间太多，主要是对命令和git的不熟悉导致的。所以还需要多多熟悉git操作，加油！！！
